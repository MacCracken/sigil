## DER parsing (untrusted input)

Both parsers reuse `der_walk` / `DER_TAG_*` / `_oid_eq` from
`src/x509.cyr` — the existing bounds-checked ASN.1 walker — rather than
a new hand-rolled parser. Every `der_walk` return is checked (`< 0`
=> reject) before the content offset is used. `_rsa_int_magnitude`
strips the leading 0x00 sign byte and **rejects** a magnitude that
exceeds the destination slot (`> out_max` => -1), so a malformed/over-
long INTEGER cannot overflow. `rsa_pubkey_from_der` accepts both bare
PKCS#1 and SPKI (discriminating on the first inner tag; SPKI checks the
rsaEncryption OID and the BIT STRING's 0 unused-bits byte).
`rsa_privkey_from_der` accepts PKCS#1 and PKCS#8 and extracts all 8 CRT
components. Tested: valid keys parse and the extracted `(n,e)` verify a
real signature; **parsed p·q == n** (bignum multiply); a truncated DER
is rejected. The private-key struct holds secret material (d, p, q, dp,
dq, qinv) — documented as caller-zeroized.

## Constant-time modexp (secret exponent)

`bn_mont_modexp` is the secret-path engine: a **square-and-multiply-
always** ladder over a **fixed** bit count (`exp_blen*8`, no leading-
zero skipping), with a constant-time `_bn_ct_select` instead of a
branch on the exponent bit — so control flow and memory-access pattern
are independent of the secret bits. CIOS Montgomery multiplication
(odd modulus required; RSA n/p/q are odd); `n0inv` via Newton's
iteration. Correctness validated by equality with the (Python-`pow`-
validated) schoolbook `bn_modexp` for small and large exponents at
256-bit and 2048-bit. The public-exponent `bn_modexp` is unchanged and
still used only for verify / verify-after-sign.

Residual (non-constant-time) paths, all on values *derived from the
public message*, not the exponent bits: the per-multiply Montgomery
final conditional subtract and `bn_sub_assign`'s borrow chain. These
are the standard targets that **base blinding** decorrelates from the
key — see Deferred.

## Signing / forgery & fault resistance

`rsa_pkcs1v15_sign_*` computes `s = m^d mod n` with the CT ladder, then
**verify-after-sign**: recomputes `s^e mod n` (public `e`, fast
`bn_modexp`) and returns failure (zeroing `s`) unless it equals `m`.
This is the Bellcore guard — a faulted modexp (especially relevant once
CRT lands) cannot leak the key via a bad signature. Signatures match an
independent Python RSA reference **byte-for-byte** (deterministic
v1.5), for SHA-256 and SHA-384, and round-trip through the 3.6.2
verifier. The EMSA-PKCS1-v1.5 encoder is now shared between sign and
verify (`_rsa_emsa_encode`), so both agree on the exact EM by
construction.

## Key zeroization

The secret is the private key (`d`, primes, CRT params) held in the
caller-provided RSAK struct — documented as caller-zeroized (matching
the rest of sigil's secret-buffer contract). The sign scratch
(`_rsa_sign_*`, `_bn_mont_*`) holds message- and signature-derived
values (EM, m, s, and powers of m), not `d` itself; not zeroized, same
as the existing modexp scratch.

## Deferred hardening (3.6.4)

- **Base blinding** — `s = (m · r^e)^d · r^-1 mod n` with a fresh random
  `r` per signature. Decorrelates the modexp-intermediate timing/power
  from the key (defends DPA and timing-*correlation*). Requires a
  modular inverse (`bn_modinv`) and a CSPRNG read; both are new and
  warrant their own focused implementation + audit. **LOW-1** tracks
  the gap.
- **CRT** — sign mod p and mod q then recombine (~4×). The
  verify-after-sign guard already in place covers the CRT fault surface
  it introduces.
- A dedicated **security audit pass** over the combined RSA surface
  ships with 3.6.4.

## Findings

- **LOW-1** — base blinding deferred to 3.6.4 (see above).
- **INFO-1** — no CRT yet (slower sign; correctness-equivalent).
- **INFO-2** — `.bss` growth from Montgomery/sign scratch; no heap, no
  per-call alloc.

Zero CRITICAL / HIGH / MEDIUM.
