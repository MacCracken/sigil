# Security Audit — 3.9.7 thread-safety banking (ECDSA / bignum / PRF / AEAD)

**Date:** 2026-06-29
**Scope:** Completion of the 3.9.6 concurrent-TLS thread-safety follow-up
(ADR 0007). Banking of the remaining per-call crypto scratch in
`src/poly1305.cyr` + `src/chacha20poly1305.cyr`, `src/ecdsa_p256.cyr` +
`src/ecdsa_p384.cyr` + `src/ecdsa_sign.cyr`, and `src/bignum.cyr` +
`src/rsa.cyr` + `src/tls12_prf.cyr`.
**Result:** every reachable concurrent crypto path is race-free. 2 findings
fixed (1 MEDIUM, 1 LOW), 1 compiler-behaviour fact documented. Suite
**1576/0 across 60 files**; two new red→green race-detectors.

---

## Findings

### F1 — MEDIUM — Latent data race in the ECDSA DER wrappers
`ecdsa_p256_sign_der`, `ecdsa_p384_sign_der`, and `ecdsa_p256_verify_der`
held their raw `r‖s` scratch in `secret var sig[64/96]` / `secret var
raw_sig[64]`. An in-code comment claimed these were per-call ("stack-local
… cyrius v6.2.25 arena fix"). **A two-thread probe disproved that:**
`secret var X[N]` ARRAYS are function-scope STATIC globals (quirk #1) that
race across threads — the arena fix only removed a per-call bump-heap leak,
it did not make them per-call thread-safe. These wrappers are exactly the
TLS 1.3 CertificateVerify path (`*_sign_der`) and mTLS client-cert verify
(`verify_der`), so concurrent handshakes could corrupt a signature being
encoded or a signature being decoded for verification (integrity / DoS;
not a key-disclosure path — `r‖s` is public).
**Fix:** banked all three (plain `var X[N*SIGIL_CRYPTO_BANKS]` + `cbank()`
lane + per-lane `memset`). Misleading comment corrected. Regression-guarded
by the DER round-trip leg of `ecdsa_concurrent.tcyr` (red on the unbanked
code, green after).

### F2 — LOW — RSA-sign secret-exponent residue left in global scratch
The RSA sign path (`_rsa_raw_sign` and the bignum Montgomery ladder) left
secret-exponent intermediates (`_bn_mont_*` = m^d / mp^dp / mq^dq,
`_rsa_blind_*`, `_rsa_crt_*`, `_bn_inv_*` blinding-factor modinv) resident
in process-global scratch after returning — only a faulty-signature discard
path wiped anything. Defence-in-depth gap (residue, not an active leak).
**Fix:** banking forces a per-lane `memset` of exactly these buffers;
`_rsa_scrub_sign_lanes()` is now called on every `_rsa_raw_sign` return
(success + both failure paths), and `bn_mont_modexp` self-wipes its mont /
`_bn_m64` lanes.

### F3 — INFO — `secret var X[N]` arrays are shared statics (documented)
Recorded as a durable fact (memory + this audit): only SCALAR `secret var x`
locals are per-call; `secret var` ARRAYS follow quirk #1 (static). Any
`secret var` array on a concurrent path must be banked as plain `var` +
per-lane `memset`, never left as `secret var` (and never banked AS
`secret var` — the whole-array wipe clobbers sibling lanes, quirk #9 /
ADR 0004). Audit of remaining `secret var` arrays: the only ones left are
KEYGEN-path (`ed25519_keypair`, mldsa/trust keygen seeds) — startup /
main-thread, not per-connection — so left as-is (out of 3.9.7 scope).

## Posture review (unchanged-or-improved)

- **Constant-time:** no new branch on secret data. The banking adds only a
  `cbank()` lane-offset (a thread-local read, input-independent) to existing
  addresses; the Poly1305 freeze and all tag/signature compares remain the
  bitwise-OR / `ct_eq_bytes` constant-time forms. KATs byte-identical.
- **Key zeroization:** every banked SECRET buffer (sign scratch, RFC 6979
  DRBG state, `k·G`, Montgomery/CRT/blind intermediates, PRF state) gets a
  per-lane `memset` on its owning return paths — verified by reading every
  scrub helper and call site. No banked array uses `secret var` (verified by
  grep: zero `secret var [name][N]` decls remain in the banked modules).
- **No new external input, syscall, or allocation-on-untrusted-size** — the
  streaming Poly1305 removes a heap allocation; nothing added.

## Verification
`ecdsa_concurrent.tcyr` (P-256+P-384 sign/verify/DER ×8 lanes) and
`bignum_tls12_concurrent.tcyr` (modexp+PRF ×8) were RED on the unbanked code
and GREEN after, stable across repeat runs. Full suite 1576/0 (60 files);
KATs (ecdsa_p256 74, ecdsa_p384 43, ecdsa_sign 20, bignum 14, rsa 38,
tls12_prf 9, poly1305 11, x509 / x509_p384 / x509_rsa) all pass; benches
within noise (RSA off-path modexp +~5%).
