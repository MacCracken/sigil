## Downstream-consumer check

Of the eight first-party consumers in `state.md`:
- **ark** — deps the local sigil dist (`path=../sigil`,
  `modules=["dist/sigil.cyr"]`). It does **not** build on this host, but
  the failure is ark-side and pre-existing: ark pins cyrius **5.1.10**
  and its vendored `lib/agnosys.cyr` is incompatible with the host cycc
  (`slice subscript requires include "lib/slice.cyr"`), erroring before
  any sigil symbol is exercised. Unrelated to sigil 3.6.x.
- **kavach** (tag 3.5.9), **phylax** (tag 3.1.1) — pin older released
  sigil tags; they adopt 3.6.x only on a deliberate pin bump.
- **daimon / aegis / argonaut** — no current sigil dependency
  (argonaut explicitly dropped it for the stdlib `ct` module);
  **mela / stiva** — not checked out on this host.

**Substantive guarantee (in lieu of a cross-repo build):** 3.6.x is
**API-additive** — every pre-3.6 public signature is preserved:
`pem_decode_privkey`, `sgx_derive_seal_key`/`sgx_seal_key`/
`sgx_unseal_key` (new `_n` variants added alongside), `rsa_pkcs1v15_
verify`/`sign`, `x509_verify_chain`, `x509_parse`, etc. The new surface
is purely additive (`rsa_pss_*`, `*_n` seal, `_ecdsa_der_int_w`,
`X509_SIG_*`/`X509_CURVE_RSA`) and the Montgomery swap is internal. A
consumer adopts 3.6.x by bumping its pin with no source migration. One
documented nuance: `pem_decode_privkey` returns `RSAK_SIZE` for RSA when
`key_max >= RSAK_SIZE`; the historical ≤48-byte scalar-buffer callers
still get the sentinel. The dist bundle self-compiles clean and sigil's
own 51-file suite is green.
