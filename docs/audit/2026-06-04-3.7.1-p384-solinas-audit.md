## Correctness — derivation-first methodology

The P-384 Solinas term table is wider and more error-prone than P-256.
Rather than transcribe a table from memory (a first attempt did, and was
caught failing immediately by the curve/verify tests), the layout was
**derived from the prime's folding relation** `2^384 ≡ 2^128 + 2^96 −
2^32 + 1 (mod p384)`: each of the 24 product words was folded into the
12 output-word positions, the coefficient matrix verified to reproduce
`x mod p` for 3000 random inputs, then decomposed into 8 positive + 3
negative ±1 terms (one positive doubled for the coeff-2 columns) and
re-verified 5000/5000 vs `x mod p`. Only then was it coded.

In-tree validation: `p384_reduce` (Solinas) equals `_p384_reduce_longdiv`
byte-for-byte over 64 SHA-384-seeded random 768-bit inputs + edges
(`2^768−1`, high-half all-ones); the existing reduce KATs, the RFC 6979
P-384 verify vectors, the on-curve / point-arithmetic / `n·G = O`
checks, and the SEV-SNP `snp_verify_full` chain all pass. Each term
`< 2^384 < 2·p384` so `_p384_sol_red1`'s single conditional subtract
reduces it into `[0, p)`; the combination uses the audited
`fp_p384_add`/`fp_p384_sub` (operands and result in `[0, p)`), giving a
fully-reduced result.

## Memory safety

`_p384_wd(prod, idx)` reads `load64(prod + (idx/2)*8)` for `idx` 0..23 →
within the 96-byte product. `_p384_mk` writes exactly 48 bytes (6
`store64`) to the 48-byte `out`/`_p384_sol_t`. `(wN << 32)` operates on
values masked to `0xFFFFFFFF` (no sign-extension). `fp_p384_add(out,
out, T)` / `fp_p384_sub(out, out, T)` self-aliasing is safe
(`u384_add`/`u384_sub` read-before-write per limb). Scratch is
single-threaded (INFO-2).

## Constant-time posture (INFO-1)

Posture-neutral, identical to the 3.7.0 finding: the Solinas reduction
has data-dependent conditional subtracts, but so did the
`_p384_long_div_reduce` it replaces (per-bit value-dependent subtract).
No new secret-dependent path. `ecdsa_p384_verify` is public-data; the
`ecdsa_p384_sign` secret-scalar exposure is pre-existing and mitigated
by RFC 6979 + single-tenant deployment.

## Performance (claims backed by `benches/history.csv`)
