## CRT signing (new)

Garner recombination: `s1 = (m mod p)^dp mod p`, `s2 = (m mod q)^dq mod
q`, `h = (s1-s2)Â·qinv mod p`, `s = s2 + qÂ·h`. Each half-width modexp
uses the **constant-time** Montgomery ladder. `s = s2 + qÂ·h < n` by
construction (`h < p â‡’ qÂ·h â‰¤ n-q`, `s2 < q`), so no final reduction.
- **Fault surface**: CRT is the classic Bellcore fault target â€” a
  glitch in one half yields an `s` that factors `n`. This is covered
  by the **verify-after-sign** guard already in the signer (recompute
  `s^e mod n`, refuse to emit unless it equals `m`); the guard runs on
  every signature, blinded and CRT'd. Validated: CRT output matches
  the Python reference byte-for-byte; a fall-back to the full-width CT
  ladder exists if a key lacks CRT parameters.

## Constant-time posture (secret exponent)

The secret-exponent exponentiations â€” full-width `bn_mont_modexp` and
both half-width CRT modexps â€” are square-and-multiply-**always**
Montgomery ladders over a fixed bit count with a constant-time select,
so control flow / memory access are independent of the exponent bits.
The value-dependent residuals identified in 3.6.3 (Montgomery final
subtract, `bn_sub` borrow chain) are now **decorrelated from the key
by base blinding** â€” the inputs to every secret modexp are randomized
per signature. The public-exponent path (`bn_modexp`, used for verify,
`r^e`, and verify-after-sign) is unchanged and carries no secret.

## Forgery / verify (unchanged, re-confirmed)

RSA verify (3.6.2) rebuilds the full expected `EM` and compares all
octets (defeats the Bleichenbacher/BERserk class), range-checks `s<n`,
exact signature length, and the 8-octet minimum `PS`. The EMSA encoder
is shared with sign, so encode and verify agree by construction.

## Key handling / zeroization

The secret is the private key (`d, p, q, dp, dq, qinv`) in the
caller-provided RSAK struct â€” documented as caller-zeroized. The
blinding/CRT scratch (`_rsa_blind_*`, `_rsa_crt_*`) holds the random
`r`/`r^-1` and **blinded-message-derived** intermediates (`mb`, `mp`,
`mq`, `s1`, `s2`), not the key; left un-zeroized, consistent with the
existing modexp scratch (**INFO-2**; `secret var`/`memset` is a
possible future defense-in-depth).
