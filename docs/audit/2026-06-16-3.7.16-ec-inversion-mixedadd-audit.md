# Sigil Security Audit — 2026-06-16 (3.7.16 P-256 verify EC-squeeze: inversion chains + mixed addition)

**Auditor**: Internal (Claude/Opus), KAT-conclusive verification
**Version audited**: 3.7.16
**Cyrius toolchain**: 6.2.12; deps agnosys 1.4.3, sakshi 2.3.0
**Prior audit**: `2026-06-15-3.7.15-windows-entropy-audit.md`
**Scope**: The P-256 `ecdsa_p256_verify` performance work (EC-squeeze levers 1 + 2a):
- **Lever 1 — modular inversions** (`src/ecdsa_p256.cyr`): the field inverse
  `fp_p256_inv` (`a^(p-2) mod p`) moved to a fixed `2^k-1`-block addition chain;
  the scalar inverse `fn_p256_inv` (`a^(n-2) mod n`) moved to a 4-bit fixed window.
  The generic binary square-and-multiply implementations are retained as
  `_fp_p256_inv_generic` / `_fn_p256_inv_generic` (KAT oracles).
- **Lever 2a — mixed addition**: new `pt_add_mixed` (Jacobian + affine,
  "madd-2007-bl"); the fixed-base `u1·G` comb table is now stored affine and its
  64 hot-path adds use `pt_add_mixed`, skipping the identity (nibble 0).

**Method**: Conclusive known-answer testing (no statistical sampling needed for the
correctness claim) + a constant-time/threat-model review of the changed paths.

## Correctness (KAT-conclusive)

The new code paths are proven correct by **implementation-independent** checks —
not just differential agreement with the prior code, but direct algebraic identities
over pseudo-random inputs (`ecdsa_p256.tcyr`, +10 assertions, full suite 1478/1478):

1. **Field inverse** — `a · fp_p256_inv(a) ≡ 1 (mod p)` over 64 random field
   elements + `a=1` and `a=p-1`. (If `a·inv ≡ 1` for random `a`, the function *is*
   the inverse — independent of any reference.) Also `chain == _fp_p256_inv_generic`
   byte-for-byte.
2. **Scalar inverse** — `a · fn_p256_inv(a) ≡ 1 (mod n)` over 64 random scalars,
   plus `window == _fn_p256_inv_generic` byte-for-byte.
3. **Mixed addition** — `pt_add_mixed(P, Q_aff) == pt_add(P, Q_jac)` (both reduced
   to affine) over 24 `(P, Q) = (k1·G, k2·G)` pairs with non-trivial Jacobian `Z`
   on `P`, plus the two `H==0` branches: `P==Q → 2P` and `P==-Q → O`.
4. **End-to-end** — the real RFC 6979 P-256/SHA-256 verify vectors and the
   tamper/wrong-key/wrong-length negative cases pass through the new affine comb +
   mixed add (`ecdsa_p256.tcyr`), and every downstream P-256 verify consumer
   (`x509`, `sgx`, `tdx`, `sev_snp`) is green.

## Security / constant-time review

- **No new secret-dependent path.** Lever 1's exponents (`p-2`, `n-2`) are **public
  constants**; lever 2a's comb is over the **public** `u1·G` term. Both run only
  inside `ecdsa_p256_verify`, which consumes exclusively public data (digest,
  signature, public key). The table-indexed comb and the window are explicitly
  documented as verify-only / non-CT (the table index would leak a nibble via cache
  timing on a *secret* scalar).
- **The secret-nonce signing path is untouched.** `k·G` for signing stays on the
  constant-time Montgomery ladder `pt_scalarmul` (`ecdsa_sign.cyr`); it does **not**
  call `p256_scalarmul_base`, `pt_add_mixed`, or the new inverses. Confirmed by
  inspection: the only callers of the new paths are the verify sites.
- **No memory-safety change.** `pt_add_mixed` reads every coordinate of `p` before
  writing any coordinate of `out` (alias-safe for the comb's `pt_add_mixed(out, out, …)`);
  the affine table is sized `64·16·64 = 65536` B with the identity entry unused;
  the inversion scratch buffers are init-once `alloc` singletons (P-256 verify is
  not on the parallel batch path, so no banking is required).
- **Shared multiply untouched.** Lever 1/2a do **not** modify `u256_mul_full` or the
  Solinas reduction; the audited shared 256-bit multiply is unchanged. (Lever 3,
  Karatsuba, which *would* touch it, is deferred to 3.7.17 with a dedicated security
  re-review — not in this release.)

## Verdict

The 3.7.16 P-256 verify speedup is **correct (KAT-conclusive) and introduces no new
side-channel or memory-safety risk**. It is verify-path-only on public data; the
constant-time signing path is unaffected. Audit floor stays empty.
