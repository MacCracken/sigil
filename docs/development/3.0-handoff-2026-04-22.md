## Gotchas the previous sessions tripped on

Write these down so the next agent doesn't re-learn them:

1. **`.tcyr` test files need explicit main invocation.** Cyrius
   doesn't auto-call `fn main`. Every test needs:
   ```
   var exit_code = main();
   syscall(60, exit_code);
   ```
   at the bottom. Without this, the test binary silently exits 0
   and all assertions are skipped — the build completes, `main`
   shows up as "dead", and you get "0 passed". Matches what
   `tests/tcyr/verify.tcyr` / `tests/tcyr/sigil.tcyr` do.

2. **"dead: main" in build output is noise.** It appears on every
   cyrius build, even for binaries that actually run (e.g.
   `build/sigil-smoke`). Don't treat it as a signal that main is
   missing.

3. **1 MB preprocessor cap (CLAUDE.md quirk #8).** Adding
   `include "src/trust.cyr"` + `src/verify.cyr` etc. to
   `tests/bcyr/sigil.bcyr` — which already pulls mldsa — pushes
   the preprocess expansion over 1 MB. The `batch_parallel.bcyr`
   file exists as a separate compilation unit specifically to
   keep under the cap. Any new bench that needs the full verify
   chain should also be its own file.

4. **16384 fixup-table cap (CLAUDE.md quirk #5).** Sigil HEAD is
   sitting immediately below the cap. Any new feature work that
   needs threading/atomic primitives OR adds meaningful
   `load64` / `store64` operations in a default-build hot path
   will overflow. The current workaround is `#ifdef` gating (see
   the `SIGIL_BATCH_PARALLEL` pattern in `src/lib.cyr` +
   `src/verify.cyr`). This is why ungating is a 3.1 item —
   cyrius upstream needs to lift the cap first.

5. **`alloc`/`vec`/`map` are NOT thread-safe (CLAUDE.md quirk #7).**
   The mutex-wrap in `sv_verify_batch` exists specifically because
   every downstream helper (`hash_file`, `hex_decode`,
   `trust_check_new`, `vresult_add_check`, `verification_result_new`,
   `map_get` on trust store) eventually hits one of these.
   Lifting the mutex requires rewriting the call chain to be
   alloc-free per-call — that's the 3.1 Option 1 work.

6. **Scope decisions belong to the user.** The previous agent
   (commit `f817821` "wierd ass agent") rewrote `sc_reduce`'s
   scratch from globals to stack-locals unilaterally, citing
   thread-safety. User kept the commit but was unhappy about
   the unilateral scope expansion. Current session learned from
   that — any plan deviation is surfaced and waits for approval.
   Memory `feedback_plan_adherence.md` encodes this rule for
   future sessions.
